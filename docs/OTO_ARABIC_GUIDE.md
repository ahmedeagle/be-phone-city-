# دليل تكامل OTO للشحن - الوضع الحالي

## 🎯 الملخص

تم تطوير تكامل كامل مع OTO API، لكن واجهنا **مشكلة في صلاحيات الحساب**.

---

## ⚠️ المشكلة

عند محاولة إنشاء طلب في OTO عبر API:

```
خطأ: HTTP 403 Forbidden
السبب: الحساب لا يملك صلاحيات إنشاء الطلبات عبر API
```

### ما يعنيه هذا

حساب OTO الحالي (مفتاح API المستخدم) من نوع:
- **StarterPackage** أو **FreePackage**
- لا يسمح بإنشاء طلبات عبر API
- يسمح فقط بإنشاء شحنات لطلبات موجودة مسبقاً

---

## 💡 الحلول المتاحة

### الحل 1: ترقية حساب OTO (موصى به) ⭐

**الخطوات:**

1. **تواصل مع دعم OTO**
   - البريد: support@tryoto.com
   - الموقع: https://app.tryoto.com

2. **اطلب تفعيل صلاحيات API**
   ```
   الموضوع: طلب تفعيل صلاحيات إنشاء الطلبات عبر API
   
   مرحباً فريق OTO،
   
   أحتاج لتفعيل صلاحية إنشاء الطلبات عبر API لحسابي.
   حالياً أحصل على خطأ 403 عند استخدام POST /orders
   
   البريد الإلكتروني للحساب: [بريدك]
   
   شكراً لكم
   ```

3. **انتظر الرد** (1-2 يوم عمل)

4. **بعد التفعيل:**
   - ✅ شحن تلقائي بضغطة زر واحدة
   - ✅ لا حاجة لخطوات يدوية
   - ✅ سرعة في العمل

---

### الحل 2: الطريقة اليدوية (يعمل الآن) ✅

**سير العمل:**

#### الخطوة 1: إنشاء الطلب في لوحة OTO
```
1. افتح https://app.tryoto.com
2. اذهب إلى "الطلبات" → "إضافة طلب"
3. أدخل:
   - معلومات العميل
   - عنوان التوصيل
   - المنتجات
   - طريقة الدفع (COD/مدفوع مسبقاً)
4. احفظ الطلب
5. انسخ "رقم الطلب" (Order ID) - مثال: 123456
```

#### الخطوة 2: في نظامك
```
1. افتح الطلب في لوحة التحكم
2. اضغط "تعديل"
3. الصق رقم طلب OTO في الحقل المخصص
4. احفظ
```

#### الخطوة 3: إنشاء الشحنة
```
1. اضغط زر "شحن مع OTO"
2. (اختياري) أضف ملاحظات للشحنة
3. اضغط "نعم، شحن الآن"
4. ✅ تم! سيتم إنشاء الشحنة تلقائياً
```

**النتيجة:**
- رقم تتبع (Tracking Number)
- رابط تتبع الشحنة
- تحديث حالة الطلب تلقائياً

## 🔧 الوضع التقني الحالي

### ما تم تطويره ✅

1. **المصادقة (Authentication)**
   - ✅ تبادل الـ Refresh Token بـ Access Token
   - ✅ تخزين مؤقت للـ Token
   - ✅ تجديد تلقائي عند انتهاء الصلاحية

2. **إنشاء الطلب (Create Order)**
   - ✅ الكود جاهز ومكتوب
   - ✅ بناء البيانات تلقائياً
   - ⚠️ محجوب بسبب صلاحيات الحساب

3. **إنشاء الشحنة (Create Shipment)**
   - ✅ يعمل بشكل كامل
   - ✅ يحتاج فقط رقم طلب OTO

4. **واجهة المستخدم**
   - ✅ زر "شحن مع OTO"
   - ✅ حقل ملاحظات اختياري
   - ✅ إشعارات نجاح/فشل
   - ✅ عرض رقم التتبع

## 🎬 كيف تستخدم النظام الآن

### للطلبات الجديدة:

1. **عندما يصل طلب جديد:**
   - غيّر حالته إلى "قيد المعالجة"
   - تأكد من وجود عنوان التوصيل

2. **اذهب إلى لوحة OTO:**
   - أنشئ الطلب يدوياً
   - انسخ رقم الطلب

3. **ارجع لنظامك:**
   - افتح الطلب
   - الصق رقم OTO
   - اضغط "شحن مع OTO"

4. **✅ تم!**
   - الشحنة ستُنشأ تلقائياً
   - رقم التتبع سيظهر
   - العميل سيستلم رابط التتبع

## ⚙️ الإعدادات المطلوبة

### ملف `.env`

```env
# معلومات OTO API
OTO_API_KEY=your_refresh_token_here
OTO_ENVIRONMENT=production
OTO_API_TIMEOUT=30

# معلومات المستودع (عنوان الإرسال)
OTO_PICKUP_NAME="اسم متجرك"
OTO_PICKUP_PHONE="966501234567"
OTO_PICKUP_EMAIL="store@example.com"
OTO_PICKUP_ADDRESS="عنوان المستودع"
OTO_PICKUP_CITY="الرياض"
OTO_PICKUP_COUNTRY="SA"
```

### كيف تحصل على API Key

1. سجّل دخول: https://app.tryoto.com
2. اذهب إلى: الإعدادات → قنوات البيع → OTO API
3. انسخ **Refresh Token**
4. الصقه في `.env` كـ `OTO_API_KEY`

## 🐛 حل المشاكل الشائعة

### المشكلة: "Order must be in 'processing' status"
**الحل:** غيّر حالة الطلب إلى "قيد المعالجة"

### المشكلة: "Missing recipient phone"
**الحل:** أضف رقم هاتف العميل في عنوان التوصيل

### المشكلة: "Order does not have OTO Order ID"
**الحل:** أنشئ الطلب في لوحة OTO أولاً والصق الرقم

### المشكلة: "HTTP 403 Forbidden"
**الحل:** هذه مشكلة صلاحيات الحساب - تواصل مع دعم OTO
