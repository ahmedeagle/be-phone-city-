### 2️⃣ نظام الشحن المتكامل (OTO Shipping Integration)

#### 🔄 دورة الشحن الكاملة:
```
طلب جديد → تأكيد الدفع → جاهزة للشحن → اختيار الفرع → إنشاء شحنة OTO
     ↓                                                          ↓
إشعار العميل                                           رقم التتبع + رابط التتبع
     ↓                                                          ↓
تحديثات تلقائية كل 15 دقيقة ← ← ← ← ← ← ← ← ←   مزامنة مع OTO API
     ↓
إشعار إيميل + إشعار موقع عند كل تغيير في الحالة
     ↓
تم التوصيل ✅ / فشل التوصيل ❌ / إلغاء الشحنة 🚫
```

#### 📊 حالات الشحن المدعومة (20+ حالة):

| الحالة | الوصف بالعربي | النوع |
|--------|--------------|-------|
| `created` | تم إنشاء الشحنة | عادي |
| `picked_up` | تم الاستلام من المستودع | عادي |
| `in_transit` | في الطريق | عادي |
| `out_for_delivery` | خرج للتوصيل | عادي |
| `delivered` | تم التوصيل ✅ | نهائي |
| `attempted_delivery` | محاولة توصيل فاشلة | فشل مؤقت |
| `delivery_failed` | فشل التوصيل | فشل دائم |
| `returned` | تم الإرجاع | فشل دائم |
| `return_to_sender` | أُعيد للمرسل | فشل دائم |
| `cancelled` | تم الإلغاء | نهائي |

#### 🏪 دعم المستودعات المتعددة:
- كل فرع يمكن ربطه بمستودع OTO محدد
- الإدمن يختار الفرع/المستودع عند إنشاء الشحنة
- القائمة المنسدلة تعرض جميع الفروع النشطة

#### ⚙️ المزامنة التلقائية:
- مزامنة حالة الشحنات كل **15 دقيقة** عبر Scheduler
- **حماية ضد التراجع**: النظام لا يرجع حالة الشحنة للخلف أبداً
- **كشف التأخير**: تنبيه يومي الساعة 09:00 لأي شحنة تجاوزت الموعد المتوقع
- **تمييز أنواع الفشل**: فشل مؤقت (إعادة محاولة) vs فشل دائم (يحتاج تدخل)

### 3️⃣ نظام الإشعارات الشامل

#### 📧 إشعارات الإيميل (6 أنواع):

| الحدث | المستلم | المحتوى |
|-------|---------|---------|
| تأكيد الطلب | العميل + الإدمن | رقم الطلب · المبلغ · تفاصيل الفرع (للاستلام) |
| تغيير الحالة | العميل + الإدمن | الحالة الجديدة · رقم التتبع · رابط التتبع · موعد التوصيل |
| جاهز للاستلام | العميل | 🎉 اسم الفرع · العنوان · الهاتف · ساعات العمل |
| فشل التوصيل | العميل + الإدمن | السبب · هل هو مؤقت أم دائم · الخطوة التالية |
| فشل إنشاء الشحنة | الإدمن فقط | رسالة الخطأ · رابط لإنشاء الشحنة يدوياً |
| تأخر الشحنة | الإدمن فقط | عدد الطلبات المتأخرة · أرقام الطلبات |

#### 🔔 إشعارات الموقع (Real-time):
- إشعارات فورية في لوحة التحكم (Filament)
- إشعارات فورية للعميل في الهيدر
- تحديث تلقائي كل **30 ثانية** بدون إعادة تحميل
- عداد الإشعارات غير المقروءة (Badge حمراء)
- خيارات: قراءة · حذف · مسح الكل

#### 📦 رابط التتبع في الإيميل:
- عند تغيير حالة الطلب إلى "تم الشحن" يُرسل إيميل يتضمن:
  - رقم التتبع
  - موعد التوصيل المتوقع
  - **زر "📦 تتبع شحنتك"** كزر رئيسي يفتح صفحة التتبع مباشرة
  - رابط "عرض الطلب" يفتح صفحة طلباتي

### 5️⃣ صفحة "طلباتي" — إعادة تصميم كاملة

#### 🎨 التصميم الجديد يشمل:

**تصفية ذكية بالتبويبات:**
- الكل · جاري التوصيل · مكتمل · ملغي — مع عدد كل فئة

**بطاقة التتبع التفاعلية** (للطلبات المشحونة):
- تصميم متدرج (Gradient) مميز
- رقم التتبع · حالة الشحنة · موعد التوصيل · آخر تحديث
- **شريط تقدم بصري** (4 مراحل): تم الطلب → تم الشحن → خرج للتوصيل → تم التوصيل
- **زر تحديث** مباشر من OTO API مع Animation
- **رابط تتبع خارجي** يفتح صفحة تتبع OTO في تبويب جديد

**تفاصيل الطلب:**
- معلومات الشحن (الاسم · العنوان · الهاتف · الملاحظات)
- جدول المنتجات (Desktop) / بطاقات المنتجات (Mobile)
- معلومات الفرع لطلبات الاستلام
- الملخص المالي (المجموع · الشحن · الخصم · الضريبة · الإجمالي)
- زر إلغاء الطلب مع نافذة تأكيد

**تجاوب كامل:**
- تصميم جديد للموبايل مع Drawer/Sheet للتفاصيل
- بطاقات مدمجة للمنتجات في الشاشات الصغيرة
